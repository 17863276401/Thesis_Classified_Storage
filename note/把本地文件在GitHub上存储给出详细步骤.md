# 把本地文件在GitHub上存储给出详细步骤

把本地文件存储到 GitHub 并设置文件夹分类，主要有两种最常用的方法：**直接在网页端操作**（适合文件少、新手）和 **使用 Git 命令行**（适合文件多、需要长期同步、开发者）。

以下是这两种方法的详细步骤：

------

### 方法一：直接在 GitHub 网页端上传（最简单）

这种方法适合你手头有一些文件和文件夹，想快速上传备份，不想安装任何软件。

#### 1. 创建仓库 (Repository)

1. 登录 GitHub。
2. 点击右上角的 **+** 号，选择 **New repository**。
3. **Repository name**: 输入仓库名字（例如 `My-Study-Notes`）。
4. **Public/Private**: 选择公开或私有。
5. 勾选 **Add a README file**（这一步建议勾选，方便初始化）。
6. 点击 **Create repository**。

#### 2. 上传文件并自动创建文件夹

GitHub 网页端支持“拖拽上传”，它可以识别你本地的文件夹结构。

1. 在你的电脑本地，整理好你的文件和文件夹。
   - *例如：你有一个名为 `Project` 的文件夹，里面包含子文件夹 `Images` 和 `Docs`。*
2. 回到 GitHub 刚才创建的仓库页面。
3. 点击文件列表右上方的 **Add file** 按钮，选择 **Upload files**。
4. **关键步骤**：将你本地整理好的**整个文件夹**（包含子文件夹的那个），直接**拖拽**到网页中间的灰色区域。
   - GitHub 会自动识别文件夹结构（例如 `Images/photo.jpg`）。
5. 等待文件上传进度条走完。
6. 在底部的 **Commit changes** 区域：
   - 第一行输入备注（例如：`上传项目文件及分类`）。
7. 点击绿色的 **Commit changes** 按钮。

> **注意**：如果通过网页端直接点击 "Choose your files" 选择文件，通常只能选文件而不能选文件夹。**拖拽文件夹**是保留目录结构的关键。

------

### 方法二：使用 Git 命令行（专业、推荐）

如果你需要经常更新文件，或者文件量很大，使用命令行是标准做法。

#### 1. 准备工作

- 确保电脑已安装 **Git** (在终端输入 `git --version` 检查)。
- 在 GitHub 上新建一个**空仓库**（创建时**不要**勾选 Add a README，保持完全空白）。

#### 2. 本地整理与初始化

1. 在电脑上创建一个总文件夹（例如 `MyProject`）。

2. 在里面建好你想要的分类文件夹，把文件放进去。

   - 结构示例：

     Plaintext

     ```
     MyProject/
     ├── Code/
     │   └── script.py
     ├── Images/
     │   └── logo.png
     └── README.txt
     ```

3. **右键**点击该总文件夹，选择 **Git Bash Here** (Windows) 或直接在终端打开该路径 (Mac/Linux)。

#### 3. 执行 Git 命令

依次输入以下命令（每行输入后按回车）：

**第一步：初始化本地仓库**

Bash

```
git init
```

*(这会在当前目录下创建一个隐藏的 .git 文件夹)*

**第二步：将所有文件（含文件夹结构）放入暂存区**

Bash

```
git add .
```

*(注意 `add` 和 `.` 之间有个空格，代表添加当前目录下所有变化)*

**第三步：提交到本地仓库**

Bash

```
git commit -m "初次提交：整理了文件夹分类"
```

**第四步：关联 GitHub 远程仓库** 复制你 GitHub 空仓库页面上的 HTTPS 或 SSH 地址（例如 `https://github.com/你的用户名/仓库名.git`）。

Bash

```
git remote add origin https://github.com/你的用户名/仓库名.git
```

**第五步：推送到 GitHub**

Bash

```
git push -u origin master
```

*(注意：如果不成功，可能是因为 GitHub 现在默认主分支名为 `main`，你可以尝试 `git push -u origin main`)*

------

### 💡 重要提示：关于“空文件夹”

GitHub 有一个特殊的机制：**它不跟踪空的文件夹**。

- **问题**：如果你在本地创建了一个叫 `Video` 的文件夹，但里面什么都没有，上传到 GitHub 后，这个文件夹**不会显示**。
- **解决方法**：如果你非要保留这个空文件夹分类，请在该文件夹内新建一个名为 `.gitkeep` 的空文件。这样 GitHub 就会认为它不是空的，从而保留这个文件夹。

------

### 总结建议

- **一次性备份**：直接用 **方法一（网页拖拽）**，简单直观。
- **长期维护/写代码**：请务必学习 **方法二（命令行）**，或者下载 **GitHub Desktop** 软件（图形化界面的 Git 工具）。
