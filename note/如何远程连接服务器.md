# 如何远程连接服务器

远程连接服务器的方式主要取决于**服务器的操作系统**（Linux 或 Windows）以及你使用的**本地设备**。

以下是三种最常用的连接方案，按推荐程度排序：

------

### 1. 连接 Linux 服务器 (最常用)

大多数用于开发、深度学习或建站的服务器都是 Linux 系统 (Ubuntu, CentOS 等)。

#### **方法 A：使用命令行 (Mac/Linux/Win10+)**

这是最快的方式，无需安装额外软件。打开终端 (Terminal) 或 PowerShell，输入以下命令：

$$\text{ssh} \quad \text{username}@\text{ip\_address} \quad -p \quad \text{port}$$

- **username**: 服务器用户名 (例如 `root` 或 `ubuntu`)
- **ip_address**: 服务器的公网 IP
- **port**: 端口号 (默认是 22，如果是 22 可以省略 `-p port`)

**示例：**

Bash

```
ssh root@192.168.1.100
# 如果端口不是默认的22，例如是20002
ssh root@192.168.1.100 -p 20002
```

输入命令后，如果是第一次连接，输入 `yes` 确认指纹，然后输入密码（**注意：输入密码时屏幕不会显示任何字符**，输完直接回车）。

#### **方法 B：使用图形化客户端 (推荐 Windows 用户)**

如果你不习惯命令行，可以使用以下工具，它们通常自带文件传输功能：

- **Xshell / Xftp**: 功能强大，个人用户免费。
- **MobaXterm**: 全能型工具，集成 SSH、SFTP、X11 转发等，非常推荐。
- **PuTTY**: 经典的轻量级工具，但功能较简陋。

------

### 2. 使用 VS Code 远程开发 (开发者首选)

如果你需要在服务器上写代码（例如跑 Python、炼丹），这是目前的最佳实践。它能让你像编辑本地文件一样编辑服务器文件。

1. **安装插件**: 在 VS Code 扩展商店搜索并安装 **"Remote - SSH"**。
2. **配置主机**:
   - 点击左下角的绿色图标 `><`。
   - 选择 `Connect to Host...` -> `Add New SSH Host...`。
   - 输入 SSH 命令 (如 `ssh root@x.x.x.x`)。
3. **连接**: 配置完成后，再次点击左下角，选择主机即可连接。

> **优势**: 直接在本地 VS Code 使用服务器的 Python 环境、调试代码、查看图片，无需频繁上传下载文件。

------

### 3. 连接 Windows 服务器

如果服务器是 Windows Server，通常使用 **RDP (远程桌面协议)**。

- **Windows 本地**:
  1. 搜索打开 **"远程桌面连接" (Remote Desktop Connection)**。
  2. 输入服务器 IP 地址。
  3. 点击连接，输入用户名（通常是 Administrator）和密码。
- **Mac 本地**:
  - 在 App Store 下载 **"Microsoft Remote Desktop"** 客户端。

------

### 4. 常见问题与准备工作

在连接之前，请确保你拥有以下信息：

1. **IP 地址**: 公网 IP（如果在同一局域网则是内网 IP）。
2. **端口号**: SSH 默认 **22**，RDP 默认 **3389**。如果是租用的云 GPU 服务器，端口通常是随机分配的（如 10022）。
3. **账号密码**: 也就是凭证。
4. **网络通畅**: 确保本地网络能访问服务器 IP，且服务器防火墙（安全组）已放行对应端口。
